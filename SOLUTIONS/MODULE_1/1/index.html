LE PRODUIT



Embarquez votre BI dans votre app

Révolutionnez votre application ! Qu'elle soit on-premise ou dans le cloud, embarquez des tableaux de bord interactifs, des graphiques modernes, un outil à la demande de création de rapports et une BI riche grâce à notre outil décisionnel Web de façon simple et riche.

La Big Data dans vos rapports

Connectez-vous directement à vos données Hadoop, MongoDB, Cassandra ou autres grâce à nos connecteurs natifs et visualisez-les. Vous pouvez ainsi créer rapports, tableaux de bord et toute votre analytique à partir de ces données sans avoir à les déplacer.

Analyses et rapports pour votre companie

Grâce à notre suite BI basée sur le Web, extensible et abordable, les utilisateurs métiers ont un accès direct aux réponses à leurs questions grâce à l’outil à la demande de création de rapports et de tableaux de bord, pendant que le déploiement et l'infrastructure sont gérés de façon optimale par les DSI.

Visualize.js

Visualize.js enrichit votre application en permettant d'embarquer de façon complètement transparente des rapports et des graphiques facilement contrôlables. Avec Visualize.js, Jaspersoft est la seule plateforme qui intègre l'analytique dans votre application en suivant les mêmes standards que vos développeurs.

<script src="http://localhost:8080/jasperserver-pro/client/visualize.js"></script>


2 - Authentification

Maintenant que nous avons référencé Visualize.js nous pouvons l'utiliser et la première étape est l'authentification. Cela se fait avec du code JavaScript que nous allons définir dans un fichier dédié.
Ouvrez le fichier assets/js/visualizejs_main.js avec votre éditeur (le fichier est normalement vide).
La première étape est de configurer la librairie avec le nom d'utilisateur et le mot de passe pour s'authentifier. Nous utiliserons la méthode basic, en production Visualize.js s'authentifiera via un token crypté. 
visualize({
    auth: {
        name: "jasperadmin",
        password: "jasperadmin",
        organization: "organization_1"
    }
}, function (v) {
	/*ICI NOUS AJOUTERONS NOTRE CODE VISUALIZE>JS*/
	alert('Visualize.js est prêt')

});

Module 2



1 - Trouver un rapport à embarquer

Ci-dessus il y a 3 emplacements, qui pour l'instant contiennent chacune une image.
Chaque image donne le nom de la balise HTML div qui le contient.
Pour commencer nous allons remplacer l'image du premier emplacement par un rapport de JasperReports Server avec Visualize.js.
1.1 - Ouvrez JasperReports Server (lien) (localhost port 8080).
1.2 - En utilisant le champ de recherche en haut à droite cherchez "Demographic Report" et puis Enter

(Un aperçut écran est disponible en passant la sourie sur les textes en bleu dans ce module)
La recherche devrait retourner un rapport ouvrez-le pour le tester.
1.3 - Maintenant revenez à la page de recherche, cliquez-droit sur le nom du rapport, puis Propriété...
Et copiez le chemin c'est le chemin dont Visualize.js à besoin

2 - Embarquez votre 1er rapport avec Visualize.js

Maintenant ajoutons le code pour intégrer notre rapport dans notre page Web!
2.1 - Ouvrez le fichier visualizejs_main.js et remplacer la ligne générant une alerte par le code ci-dessous. Vérifier que l'option resource pointe vers notre rapport (même chemin). 

v("#div_chart_1").report({
	resource: "/public/Samples/Reports/02._Sales_Mix_by_Demographic_Report",
	scrollToTop:false,
	scale: "height",
	error: function (err) {
		alert(err.message);
	}
});
2.2 - Sauvegardez le fichier et raffraissez cette page! Notre rapport est intégré à l'emplacement choisi (un peu plus haut) tout simplement!


3 - Ajouter un graphique avec son intéractivité

Maintenant nous allons remplacer les 2 autres emplacement (Chart 2 et Chart 3) par 2 autres graphiques.
Une fois les appels Visualize.js rajoutés seulement le 2ème graphique s'affichera,
le 3ème sera controlé par le 2ème en cliquant sur une portion.
Premièrement ajoutons le 1er graphique, en générant seulement une alerte en cliquant sur une portion.
Ajoutez le code qui suit juste après le code ajouté précédemment dans visualizejs_main.js: 

v("#div_chart_2").report({

	linkOptions: {

			"mouseOver":function (ev, link) {},

			"click": function (ev, link) {				
				 if (link.type == "ReportExecution") {
					 alert('Vous avez cliqué sur l'État: '+ link.parameters.store_state);

				}
			}
		}
	},
	error: function (err) {
		alert(err.message);
	}
});

4 - Ajout de la sélection en contrôle d'entrée pour le rapport

Notre rapport à donc un contrôle d'entrée qui prend la valeur de la famille de produit à afficher: Jaspersoft est conscient de la liste de valeurs disponibles.
Nous allons donc maintenant afficher la liste de ces valeurs dans un menu déroulant. 

var inputControls = v.inputControls({
	resource: "/public/Samples/Reports/06g.ProfitDetailReport",
	success: function(data){
		var productFamilyInputControl = _.findWhere(data, {
			id: "ProductFamily"});
			
		_.each(productFamilyInputControl.state.options, function (option) {
			$("#productFamilySelector").append("<option " + 	
				(option.selected ? "selected" : "") + 
				" value='" + option.value + "'>" + 
				option.label + 
				"</option>");
		});
	}
});

$("#productFamilySelector").on("change", function () {
	drinkFoodReport.params({
		"ProductFamily": [$(this).val()]
	}).run();
});

2 - Récupérer la liste des tableaux de bord de JRS

Le code ci-dessous envoie une requête Visualize.js afin de récupérer la liste des tableaux de bord dans un dossier spécifique. Remarquez la fonction appelé pour Success, listRepository, c'est une fonction que nous n'avons pas encore définie, ce sera l'object de la prochaine étape.
Donc procédez directement à la prochaine étape (3) avant de raffraichir la page 

v.resourcesSearch({
	folderUri:"/public/Samples/Dashboards",
	recursive:false,
	types:["dashboard"],
	success: listRepository,
	error: function (err) {
		alert(err.message);
	}
});


3 - Remplissage du menu déroulant et affichage du tableau de bord

cette dernière étape va nous permettre de parser le résultat (JSON) de notre recherche (étape précédente).
Le menu déroulant sera rempli avec cette liste. 

// Création de la liste depuis le référenciel
function listRepository(results) {
	var dSelector = $("#dashboardSelector");
	dSelector.empty(); // enlève les options existante
	$.each(results, function() {
		dSelector.append($("<option></option>")
		 .attr("value", this.uri).text(this.label));
	});

	//console.log($("#select"));
	dSelector.change(function() {
		v.dashboard({
				resource: this.value,
				container: "#div_dashboard_1",
				error: function (err) {
					alert(err.message);
				}
		});
	});
	// Utilisation d'un index pour selectionner une option à afficher par défaut.
	$("#dashboardSelector option:eq(5)").prop("selected", true).change(); 
}

Gérer la navigation forcée en haut de page

À chaque fois qu'un tableau de bord est affiché, l'affichage revient au haut de page.
Cela est dû à la propriété scrollToTop qui n'est pas exposé dans l'API.
Ce comportement peut être modifier via le fichier de configuration de JasperReports Server.
Cela se fait dans le fichier suivant:
/apache-tomcat/webapps/jasperserver-pro/scripts/bower_components/bi-report/src/bi/report/view/ReportView.js
Changez la ligne 195 en remplaçant le true par false, la ligne deviendra donc comme ci-dessous.
Notez que le cache du navigateur web doit être vidé (Ctrl+Shift+delete pour Chrome) pour prendre en compte le changement.
Notez aussi que cela n'affectera que le code de Visualize.js en mode compressé (paramètre _opt=false dans l'url de Visualize.js en haut de la page index.html). 

scrollToTop = typeof this.stateModel.get("scrollToTop") === "undefined" ? false : this.stateModel.get("scrollToTop"),
